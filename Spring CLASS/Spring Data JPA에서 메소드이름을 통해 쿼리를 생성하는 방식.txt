* Spring Data JPA에서 메소드이름을 통해 쿼리를 생성하는 방식 *

- 메소드의 이름이 엔티티 클래스의 필드 이름과 일치해야 한다는 규칙에 기반함.
- 이를 통해 JPA는 findBy 뒤에 오는 문자열을 분석하여 자동으로 SQL 쿼리를 생성함.

=>  따라서, 메소드 이름과 필드 이름이 정확히 일치해야 하며, 필드에 적절한 JPA 어노테이션을 적용해야 함.

- 확인 방법

1. 엔티티 클래스의 필드 이름 확인 :
   -> 필드의 이름이 메소드 이름과 정확히 일치하는지 확인함.

2. @Column 어노테이션 확인 :
   -> @Column 어노테이션의 name 속성이 데이터베이스의 컬럼명과 일치하는지 확인함. 
   -> ex ) 
	@Entity
	public class LectureCategory {
 	   // ...

	    @Column(name = "lecture_category_name") // 데이터베이스 컬럼명과 일치해야 함.
    	private String lectureCategoryName;

	    // ...
	}

3. 리포지토리 인터페이스의 메소드 이름 확인 :
   -> Repository 인터페이스에 해당 메소드가 정의되어 있는지 확인.
   -> 메소드 이름이 @Column 어노테이션이 적용된 해당 필드를 기준으로 쿼리를 생성할 수 있도록 구성되어 있는지 확인.

4. 프로젝트 재빌드 및 실행 :
   -> 변경사항을 적용한 후에는 변경사항을 저장하고 프로젝트를 재빌드함.
   -> Maven을 사용하는 경우 mvn clean install, Gradle을 사용하는 경우 ./gradlew clean build 명령어를 사용할 수 있음.
