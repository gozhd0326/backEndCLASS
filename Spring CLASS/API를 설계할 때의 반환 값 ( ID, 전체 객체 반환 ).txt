* API를 설계할 때의 반환 값( ID , 전체 객체 반환 ) *

1. ID만 반환하는 경우 : 

- 이 경우는 주로 생성, 수정 등의 작업을 수행한 후에 그 결과를 확인할 필요가 있을 때 사용됨.
 예를 들어, 사용자가 장바구니에 상품을 추가한 후에 그 상품을 관리할 필요가 있는 경우,
 해당 상품의 ID를 반환하여 사용자가 그 상품을 추적하거나 관리할 수 있게 함.

- 또한, 서버의 부하를 줄이는 데 도움이 됩니다.
 전체 객체를 반환하면 그 객체의 모든 필드를 데이터베이스에서 읽어와야 하지만,
 ID만 반환하면 그럴 필요가 없습니다.


2. 전체 객체를 반환하는 경우

- 이 경우는 주로 조회 작업을 수행한 후에 그 결과를 사용자에게 보여주어야 할 때 사용됨.
 예를 들어, 사용자가 자신의 장바구니를 조회하는 경우,
 서버는 사용자의 장바구니에 담긴 모든 상품의 정보를 반환하여 사용자가 그 정보를 볼 수 있게 함.

-  또한, 사용자가 서버에 추가적인 요청을 보낼 필요를 줄이는 데 도움이 됩니다.
 예를 들어, 사용자가 장바구니에 상품을 추가한 후에 그 상품의 상세 정보를 보려면,
 상품의 ID를 받아서 다시 서버에 요청을 보내야 합니다.
 하지만 서버가 처음에 상품의 전체 정보를 반환하면 그럴 필요가 없습니다.
