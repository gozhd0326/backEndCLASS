* delete() 메소드와 deleteById() 메소드 *

- delete()와 deleteById() 메소드는 Spring Data JPA에서 제공하는 메소드로,
  둘 다 엔티티를 삭제하는 기능을 수행함.
 그러나 이 두 메소드는 인자와 동작 방식에서 약간의 차이가 있음.

1. delete(T entity) :

  - 이 메소드는 삭제할 엔티티 객체를 직접 전달받아 해당 엔티티를 삭제함.

  - 삭제를 수행하기 전에 EntityManager가 해당 엔티티를 영속성 컨텍스트에서 관리하는지 확인하고,
    관리되지 않는다면 영속성 컨텍스트에 먼저 영속화합니다.
    그 후 삭제를 수행합니다.
 
 - ex ) 
 	ClassRoom classRoom = classRoomRepository.findById(classRoomId)
                .orElseThrow(() -> new RuntimeException("ClassRoom not found"));
classRoomRepository.delete(classRoom);


2. deleteById(ID id) :
  - 이 메소드는 삭제할 엔티티의 ID를 전달받아 해당 엔티티를 삭제함.

  - EntityManager가 영속성 컨텍스트에서 엔티티를 관리하고 있는지 확인하지 않고,
    바로 삭제 쿼리를 실행함.

  -   따라서 deleteById()는 delete()에 비해 성능상의 이점이 있을 수 있음.

  - ex )
	classRoomRepository.deleteById(classRoomId);


=> 결과적으로 두 메소드 모두 동일한 결과를 가져오지만,
 delete()는 삭제할 엔티티 객체를 먼저 영속화하는 추가적인 작업이 필요하며,
 deleteById()는 이러한 작업 없이 바로 삭제 쿼리를 실행함.
